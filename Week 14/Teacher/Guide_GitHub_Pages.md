# คู่มือการสอน: การนำเว็บไซต์ขึ้นออนไลน์ฟรีด้วย GitHub Pages

ในสัปดาห์นี้ เป้าหมายคือการให้นักเรียนนำโปรเจกต์ **Personal Wiki** จากสัปดาห์ที่ 3 ขึ้นไปออนไลน์บนอินเทอร์เน็ตจริงๆ เพื่อให้เพื่อนๆ และครูสามารถเข้าชมได้ผ่าน URL

## 🎯 กิจกรรมหลัก (Core)
ให้นักเรียนอัปโหลดโปรเจกต์สัปดาห์ที่ 3 ขึ้นเว็บจริง โดยใช้ GitHub Pages

### ขั้นตอนการทำ (สำหรับสอนนักเรียน):
1. **สมัครสมาชิก GitHub**: หากนักเรียนยังไม่มีบัญชี ให้สมัครที่ [github.com](https://github.com/)
2. **สร้าง Repository ใหม่**:
   - กดปุ่ม **New** หรือเครื่องหมาย **+** ที่มุมขวาบน
   - ตั้งชื่อ Repository (เช่น `my-portfolio` หรือ `personal-wiki`)
   - เลือกเป็น **Public**
   - กด **Create repository**
3. **อัปโหลดไฟล์**:
   - กดที่ลิงก์ **uploading an existing file**
   - ลากไฟล์ `index.html`, `about.html`, `hobbies.html` (จาก Week 3) มาวาง
   - กด **Commit changes**
4. **เปิดใช้งาน GitHub Pages**:
   - ไปที่แท็บ **Settings** ของ Repository นั้น
   - เมนูด้านซ้าย เลือก **Pages**
   - ในหัวข้อ **Branch** ให้เลือกเป็น `main` หรือ `master` และโฟลเดอร์ `/ (root)`
   - กด **Save**
5. **รอระบบประมวลผล**: ประมาณ 1-2 นาที ระบบจะแสดงลิงก์สีเขียวด้านบน (เช่น `https://username.github.io/repository-name/`) สามารถกดเข้าไปดูเว็บจริงได้เลย!

## 🚀 กิจกรรมเพิ่มเติม (Extra Challenge)
**"สลับกันหาลิงก์พัง (Broken Link) ของเพื่อน"**
- ให้นักเรียนจับคู่กัน หรือแชร์ลิงก์ลงในกลุ่ม
- ทดลองกดลิงก์เมนูต่างๆ ในเว็บของเพื่อนว่าเชื่อมโยงไปหน้าอื่นได้ถูกต้องไหม
- หากเจอหน้า 404 Not Found (ลิงก์พัง) ให้ช่วยกันดูว่าตั้งชื่อไฟล์หรือเขียน Path ผิดตรงไหน (เช่น ลืมใส่ `.html` หรือตัวเล็กตัวใหญ่ไม่ตรงกัน)
